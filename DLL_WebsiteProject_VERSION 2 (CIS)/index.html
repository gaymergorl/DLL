    About this project

    This website is a student-built cybersecurity self-evaluation created to help small organizations think about their current cybersecurity practices. It is for educational use and does not replace a professional security assessment.

    Why the focus on SMEs under 50 employees

    This project focuses on organizations with fewer than 50 employees, which aligns with the European Union definition of micro and small enterprises. Companies in this size range often face resource constraints, such as limited time, budget, and access to cybersecurity expertise.

    Security Functions Used in This Assessment

    This assessment is based on selected safeguards from the CIS Critical Security Controls v8 (Implementation Group 1). The results are summarized using six widely recognized cybersecurity functions: Govern, Identify, Protect, Detect, Respond, and Recover.

    GOVERN

    Establish responsibility, policies, and oversight.

    IDENTIFY

    Understand assets, data, and risks.

    PROTECT

    Apply safeguards such as MFA, patching, and access control.

    DETECT

    Identify suspicious activity and potential incidents.

    RESPOND

    Take action during cybersecurity incidents.

    RECOVER

    Restore operations and improve resilience.

    Framework simplification for SMEs

    This assessment is based on selected safeguards from the CIS Critical Security Controls v8 – Implementation Group 1 (IG1). IG1 represents a baseline set of cybersecurity practices designed for small organizations with limited cybersecurity resources. The results are summarized across six cybersecurity functions (Govern, Identify, Protect, Detect, Respond, Recover) and visualized in a radar chart.
